# Backend

FastAPI-сервис (порт 8000): приём документов, конвейер обработки, вопросно-ответный
оркестратор и веб-интерфейс «Графа». Полное описание системы, таблицы API
и переменных окружения — в [корневом README](../README.md).

## Модули

| Файл | Назначение |
| --- | --- |
| `app/main.py` | HTTP-эндпоинты и composition root: собирает `ApplicationStore` с адаптерами PostgreSQL/Neo4j/MinIO, очередь загрузок, `QueryOrchestrator`; монтирует `/ui` |
| `app/storage.py` | `ApplicationStore` — конвейер инжеста и рабочее состояние приложения в памяти процесса (документы, фрагменты, кандидаты, факты, векторы); выбор провайдеров по переменным окружения |
| `app/persistence.py` | `PostgresSink` (персистентная копия состояния, восстановление при старте, идемпотентные `ALTER` в `ensure_schema`) и `Neo4jSink` (проекция графа знаний) |
| `app/file_storage.py` | Адаптер MinIO: оригиналы документов и их PDF-превью |
| `app/jobs.py` | Очередь фоновой обработки загрузок; статусы задач в памяти, персистентная копия в таблице `jobs` |
| `app/schemas.py` | Pydantic-модели API и домена; `ONTOLOGY_LABELS` — единственный источник меток Neo4j |
| `app/ml_mock.py` | Заглушка сервиса извлечения (порт 8001; используется базовым compose без override) |

### app/pipeline/

| Файл | Назначение |
| --- | --- |
| `query.py` | `QueryOrchestrator`: сбор доказательной базы (граф и векторный поиск), генерация ответа, перевод номерных цитат в канонические идентификаторы, фильтр нерелевантных вопросов |
| `query_parsing.py` | Разбор вопроса в структурированный запрос средствами LLM: материал, процесс, условия, сущности онтологии |
| `parsers.py` | Парсеры форматов: PDF (включая рендер сканов в PNG), DOCX, PPTX, XLSX/CSV, TXT/MD/JSON; нераспознанное содержимое регистрируется фрагментом-заглушкой |
| `document_traits.py` | Признаки документа: тип и научность определяет LLM по титульным страницам с обоснованием-цитатой; происхождение и год — эвристики без LLM, при расхождении приоритет у имени файла |
| `office_render.py` | PDF-превью DOCX/DOCM/PPTX через LibreOffice; при сбое превью не создаётся, документ остаётся доступен текстом |
| `providers.py` | Провайдеры извлечения и эмбеддингов: удалённые (ml-extraction) и автономные (заглушка, hash-эмбеддер) |
| `validation.py` | Пороги кандидатов и диапазоны правдоподобия чисел из `domain/default/validation-rules.yaml` |
| `normalization.py` | Канонизация текста, идентификаторы, направления эффекта, единый список недопустимых подписей — функции, общие для слоёв запросов и хранения |
| `llm_bridge.py` | HTTP-мост к LLM через ml-extraction (`/chat_json`, `/chat_stream`); каскад провайдеров реализован на стороне сервиса |
| `dyk.py` | Рубрика «Знаете ли вы?»: LLM-формулировка фактов с кэшем и фоновым прогревом |
| `sample_data.py` | Демонстрационные документы и факты |

## Веб-интерфейс (ui/)

Один файл `ui/index.html` — одностраничное приложение без сборки и внешних
зависимостей: маршрутизация по hash, данные API вставляются только через
`textContent`/`createElement`. Просмотрщик pdf.js в репозитории не хранится:
`backend/Dockerfile` устанавливает его при сборке образа в `ui/pdfjs/`.

## Миграции

`migrations/001_init.sql` применяется однократно, при первом создании тома
PostgreSQL (`docker-entrypoint-initdb.d`). Последующие изменения схемы выполняет
`PostgresSink.ensure_schema` идемпотентными `ALTER` при старте сервиса; новые
файлы миграций не добавляются.

## Скрипты

- `scripts/seed_sample_data.py` — загрузка демонстрационных данных.
- `scripts/reprocess_documents.py` — повторная обработка загруженных документов;
  конфигурация — те же переменные окружения, что у сервиса.

## Тесты

14 файлов, 162 теста (`unittest`, запуск через pytest); сеть и LLM замещены
заглушками. Запуск в отдельном контейнере:

```bash
docker compose run --rm --no-deps -T \
  -v "$PWD/backend/tests:/app/backend/tests" \
  backend python -m pytest tests -q
```

Локально: `cd backend && python -m pytest tests`.
