# domain/ — доменная конфигурация

Знания о предметной области, вынесенные из кода: онтология, словари, единицы
измерения, правила валидации и промпты. Каталог монтируется read-only в оба
сервиса (backend — `/app/domain`, ml-extraction — `/domain`); изменения
применяются перезапуском контейнеров, без пересборки образов.

## Файлы (default/)

| Файл | Назначение | Потребитель |
| --- | --- | --- |
| `ontology.yaml` | Онтология горно-металлургического R&D (`rnd-metallurgy-v1`): типы сущностей, их свойства и примеры | ml-extraction (промпт извлечения), backend (разбор вопроса) |
| `synonyms.csv` | Канонизация терминов `canonical,alias,type`: русские и английские варианты материалов, свойств, процессов | backend (нормализация фактов и вопросов) |
| `units.yaml` | Единицы измерения: каноническая форма и алиасы (°C, ч, %, МПа и др.) | backend (сверка чисел факта с текстом источника) |
| `validation-rules.yaml` | Пороги кандидатов (auto_approve 0.85, review_min 0.60) и физические диапазоны числовых параметров | backend (валидация и автоутверждение) |
| `extraction-schema.json` | JSON Schema кандидата факта: обязательные поля, допустимые направления эффекта | ml-extraction (валидация ответа LLM) |
| `query-templates.yaml` | Шаблоны типовых вопросов: сравнение экспериментов, поиск пробелов | backend |
| `ranking.yaml` | Веса ранжирования выдачи: совпадение материала, свойства и др. | backend |
| `prompts/extraction.ru.md` | Правила извлечения для LLM: запрет вымысла, обязательный источник, гипотезы отдельно | ml-extraction |

Пороги и диапазоны в `validation-rules.yaml` и `units.yaml` — зона инженера
знаний: значения меняются без участия разработчиков и читаются при старте сервисов.
